Frank

DESIGN+DEVELOPMENT, WEBSITE (MOBILE VIEW ONLY)

Frank is social network application designed to help users improve their fashion. Users can post photos of their clothes and recieve brutally honest critiques in the form of either upvotes or comments.

At a Glance —

  • Tech Used: HTML, CSS, JavaScript, Node.js, Photoshop, Illustrator, Bootstrap

  • Main Takeaway: It's difficult to know what users want. Needfinding is extremely useful for constructings apps that solve novel problems in the most human-friendly way.

  • Team Demographics: 3 Developers

A More Detailed Look —

The Goal

Our goal was to build an mobile site that had transactional functionality. We were given the constraints that it must be a website that acted like a mobile app, it had some sort of database-related functionality, and that the features would come from extensive needfinding.

Given these constraints, and based on prelimiary needfinding, we decided a clothing-critique app met the requirements for the the project.

The Takeaway

Frank was the first HCI-focused application I worked on. I learned about the importance of user testing and needfinding. There were a few features we included, such as anonymous login, due to feedback we got from users. There were some improvements we could have made, for instance we could have enhanced the user profiles, however we ran out of time.
